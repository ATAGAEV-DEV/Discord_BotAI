## 🚀 Что умеет этот бот

### 1. 💬 Общение с ИИ
Основная функция бота. Если ваше сообщение начинается с `!` (восклицательный знак), бот воспринимает это как обращение к нему, анализирует контекст и отвечает.

*   **Пример**: `!Привет, как дела?` или `!Расскажи про Python`.
*   **Контекст**: Бот помнит нить разговора, так что можно задавать уточняющие вопросы.
*   **Инструменты**: Умеет искать актуальную информацию в интернете и проверять погоду.

### 2. 🎭 Развлечения
*   **!toxic**: Режим "прожарки" чата. Едкие комментарии в разных стилях (бабка, гопник и др.).
*   **Ранги**: Система уровней за активность. Пишите сообщения — получайте красивые карточки рангов.
*   **Поздравления**:
    *   **Дни рождения**: Автоматически запоминает даты (`!birthday`) и генерирует уникальные поздравления.
    *   **Праздники**: Каждую ночь (в 00:01 МСК) проверяет календарь праздников и поздравляет сервер с главными событиями дня.

### 3. 🔌 Полезные интеграции
*   **YouTube**: Моментальные уведомления о выходе новых видео с любимых каналов.
*   **RAG (Память)**: Бот знает информацию о пользователях сервера и использует её в общении.

## 🛠 Команды бота

### 👤 Пользовательские команды
| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `!help` | - | Показать интерактивное меню помощи |
| `!rank` | `[list]` | Ваш текущий прогресс или список всех доступных рангов |
| `!birthday` | `[DD.MM]` | Сохранить дату рождения для AI-поздравлений |
| `!toxic` | `list` | Показать список доступных персонажей для прожарки |
| `!toxic` | `[count] [persona]` | Прожарка чата. `count` - кол-во сообщений (def: 20), `persona` - стиль. |
| `!desc_list` | - | Показать все описания пользователей сервера |

### 🛡 Команды Администратора
| Команда | Аргументы | Описание |
| :--- | :--- | :--- |
| `!holiday` | `[DD.MM] [Название]` | Добавить или обновить праздник (например, `!holiday 01.01 Новый Год`) |
| `!check_holiday` | - | Принудительная проверка праздников |
| `!update_user`| - | Переиндексировать пользователей сервера для RAG |
| `!reset` | - | Полная очистка контекстной истории сервера |
| `!check_birthday`| - | Принудительная проверка и отправка поздравлений |
| `!desc_add` | `[ник] [описание]` | Добавить или обновить описание пользователя |
| `!desc_remove` | `[ник]` | Удалить описание пользователя |
| `!desc_reload` | - | Перезагрузить кэш описаний из базы данных |

### 📺 Настройка YouTube уведомлений

Для настройки уведомлений о новых видео используйте следующие команды:

#### 1. Добавление канала (`!add_youtube`)
Команда используется для первичного добавления YouTube-канала в список отслеживаемых.

**Синтаксис:**
`!add_youtube <youtube_id> <discord_channel_id> <name>`

**Аргументы:**
*   `youtube_id`: Уникальный идентификатор канала YouTube (обычно начинается на `UC`, можно взять из ссылки на канал).
*   `discord_channel_id`: ID текстового канала вашего сервера, куда бот будет присылать уведомления.
*   `name`: Короткое имя (псевдоним) для удобного управления каналом в дальнейшем (например `memes`).

#### 2. Управление уведомлениями (`!youtube`)
Включение или отключение уведомлений для уже добавленного канала по его имени.

**Синтаксис:**
`!youtube <on/off> <name>`
*   `on/off`: Включить (`on`) или выключить (`off`) уведомления.
*   `name`: То самое имя, которое вы задали при добавлении канала.

## 🏗 Технологический стек

*   **Ядро**: `Python 3.13+` & `discord.py` (Cogs architecture)
*   **ИИ Движок**: `LlamaIndex` (RAG), `OpenAI SDK`
*   **Базы данных**: 
    *   **PostgreSQL**: Основное хранилище (SQLAlchemy + asyncpg)
    *   **ChromaDB**: Векторная база данных для семантического поиска
*   **Инфраструктура**: `Docker`, `uv` (пакетный менеджер), `Ruff` (линтинг)
*   **Протоколы**: `MCP (Model Context Protocol)` для взаимодействия с внешними инструментами

## 📥 Полное руководство по установке

Следуйте этой инструкции шаг за шагом, чтобы запустить собственного бота.

### Шаг 1: Скачивание и подготовка

1.  Убедитесь, что у вас установлены [Docker](https://www.docker.com/) и [Git](https://git-scm.com/).
2.  Склонируйте репозиторий проекта:
    ```bash
    git clone https://github.com/ATAGAEV95/Discord_BotAI.git
    cd Discord_BotAI
    ```
3.  Создайте файл конфигурации из шаблона:
    ```bash
    cp .env.example .env
    ```

### Шаг 2: Настройка конфигурации (.env)

Откройте файл `.env` в любом текстовом редакторе. Этот файл содержит все секретные ключи и настройки.

**Справочник всех переменных:**

| Категория | Переменная | Описание |
| :--- | :--- | :--- |
| **Core** | `DC_TOKEN` | Токен бота из [Discord Developer Portal](https://discord.com/developers/applications). |
| | `DATABASE_URL` | URL PostgreSQL (формат: `postgresql+asyncpg://user:pass@host:port/db`). |
| **AI** | `AI_PROVIDER` | Активный провайдер: `aitunnel` или `polza`. |
| | `AI_MODEL` | Модель ИИ (например, `claude-haiku-4.5` или `gpt-4o-mini`). |
| | `AI_MODEL_MINI` | "Младшая" модель для простых задач (по умолчанию `gpt-5-mini`). |
| | `AI_TOKEN1` | Ключ доступа для выбранного AI провайдера. |
| **Services** | `WEATHER_API` | Ключ [OpenWeatherMap](https://openweathermap.org/api) для погоды. |
| | `SEARCH_API` | Ключ [Tavily](https://tavily.com/) для поиска информации в вебе. |
| | `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота для отправки отчетов администратору. |
| | `TELEGRAM_CHAT_ID` | ID чата в Telegram для получения уведомлений. |

---

### Шаг 3: Выбор режима работы

Выберите один из двух вариантов настройки `.env` в зависимости от ваших задач.

#### 🟢 Вариант 1: «Минимальный» (Только ИИ)
*Идеально для быстрого старта. Только общение, без лишних сервисов.*

1.  В `.env` заполните только секции **Core** и **AI**.
2.  В файле [main.py](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/main.py) отключите внешние интеграции:
    ```python
    ENABLE_TELEGRAM_NOTIFIER = False
    ENABLE_WEATHER = False
    ENABLE_SEARCH = False
    ```

#### 🔵 Вариант 2: «Полный фарш» (Все функции)
*Максимальные возможности: погода, поиск в интернете, отчеты в Telegram.*

1.  В `.env` заполните **ВСЕ** переменные, включая ключи для поиска и погоды.
2.  В файле [main.py](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/main.py) убедитесь, что интеграции включены:
    ```python
    ENABLE_TELEGRAM_NOTIFIER = True
    ENABLE_WEATHER = True
    ENABLE_SEARCH = True
    ```

### Шаг 4: Тонкая настройка (Лимиты и Промты)

Перед запуском вы можете настроить поведение бота под свой сервер.

#### 1. Настройка лимитов ([main.py](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/main.py))
Откройте файл [main.py](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/main.py) и найдите блок настроек:

```python
# Лимиты
CONTEXT_LIMIT = 50      # Сколько последних сообщений бот помнит в разговоре
REPORT_MSG_LIMIT = 15   # Через сколько сообщений генерировать отчет
REPORT_TIME_LIMIT = 60  # Тайм-аут (минуты) для отправки отчета
```

#### 2. Настройка личности (`app/tools/prompt.py`)
В этом файле вы можете изменить:
*   **Эмодзи сервера** (`EMOJIS`) — добавьте свои айдишники эмодзи.
*   **Описания пользователей** (`USER_DESCRIPTIONS`) — расскажите боту, кто есть кто на сервере.
*   **Система рангов** (`RANK_CONFIG`) — настройте названия, пороги и дизайн рангов.

*(Подробнее см. раздел [Настройка персонализации](#-настройка-персонализации-промты-и-эмодзи) ниже)*

### Шаг 5: Сборка и запуск

Теперь, когда все настроено, запустите бота через Docker.

1.  **Сборка образа**:
    ```bash
    docker build -t discord-bot-ai .
    ```

2.  **Запуск контейнера**:
    ```bash
    docker run -d --restart=always --name discord-bot-ai --env-file .env discord-bot-ai
    ```

🎉 **Готово! Бот должен появиться в сети.**

## 🎨 Настройка персонализации (Промты и Эмодзи)

Бот поддерживает гибкую настройку под атмосферу конкретного сервера. Основные настройки находятся в файле [`app/tools/prompt.py`](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/app/tools/prompt.py).

### 😃 Настройка Эмодзи

Эмодзи используются ботом для придания эмоциональной окраски ответам.

1.  **Поиск файла**: Откройте [`app/tools/prompt.py`](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/app/tools/prompt.py).
2.  **Структура данных**: В начале файла находится список `EMOJIS`, содержащий объекты класса `Emoji`.

```python
EMOJIS = [
    Emoji("yoba", "1101900451852599427", "издевательство")
]
```

Где:
*   `"yoba"` — Название эмодзи, используется ботом (например `:yoba:`).
*   `"1101900451852599427"` — ID эмодзи в Discord.
*   `"издевательство"` — Описание эмодзи, чтобы ИИ понимал контекст использования.

### 👥 Описания пользователей

Бот может узнавать пользователей и учитывать их особенности при общении или поздравлениях. Теперь описания хранятся в базе данных и кэшируются в памяти для мгновенного доступа. Управление происходит прямо из Discord:

*   **Добавить/Обновить**: Используйте команду `!desc_add <ник> <описание>` (только для админов).
    *   *Пример*: `!desc_add atagaev Арби, создатель бота`
*   **Удалить**: Команда `!desc_remove <ник>` (только для админов).
*   **Просмотреть список**: Команда `!desc_list` покажет все текущие описания на сервере.

Все изменения мгновенно применяются в памяти бота и используются для генерации ответов ИИ, поздравлений с днем рождения и "прожарок".

### 🎖 Настройка системы рангов

Вы можете полностью настроить систему уровней: названия, пороги сообщений, цвета и фоны карточек.

1.  **Конфигурация рангов**: В файле [`app/tools/prompt.py`](https://github.com/ATAGAEV95/Discord_BotAI/blob/main/app/tools/prompt.py) найдите список `RANK_CONFIG`.
2.  **Формат**:
    ```python
    RANK_CONFIG = [
        {
            "name": "Новичок",        # Название ранга
            "threshold": 0,           # От скольки сообщений выдаётся
            "next_threshold": 50,     # Порог для следующего уровня
            "color_name": "light_grey", # Цвет (из COLOR_MAP в utils.py)
            "text_color": (130, 130, 130), # RGB цвет текста на карточке
            "bg_filename": "rang0.jpg",    # Файл фона в app/resource/
        },
        # ... остальные ранги
    ]
    ```

Изменения в этом списке автоматически обновят:
*   Текстовую команду `!rank list`
*   Логику выдачи рангов
*   Генерацию карточек (картинка, цвет, прогресс-бар)
